RZen Rental — Owner app flow

Living reference for the Next.js app · Same typeface as prototype-single.html

Audience: You and your mother running a small apartment. No public listings — the app is owner-only after sign-in. For a plain Markdown version, see app-flow.md in this folder.

High-level routing

  • / redirects to /dashboard.
  • All /dashboard/* routes require Supabase Auth (middleware → /login?next=… when env is configured).
  • If Supabase env vars are missing locally, middleware does not enforce login (useful for dry runs).

App map (top tabs)

Tab Route Purpose
Dashboard /dashboard Room overview grid: status pill, monthly rate, Mark occupied / Mark vacant, ⋮ menu for lease summary and all status options.
Payments /dashboard/payments Stat cards, By billing month chart, add payment form, transactions table, Mark paid.
Tenants /dashboard/tenants All leases rows (active + ended), newest lease start first.
Utilities /dashboard/utilities Electric / water readings per room and period → utility_readings.
Sign in /login Supabase email/password; signed-in users hitting login redirect to dashboard (or next).

The dashboard home header also links to Payments, Tenants, and Utilities for quick jumps.

Flow 1 — Room lifecycle (Dashboard)

Mark a unit occupied

  1. Open Dashboard.
  2. Find a Vacant card.
  3. Click Mark occupied (green).
  4. Updates rooms.status to occupied via your session (RLS).
Tenant name and lease terms are not collected on that button. Add or edit the active lease in Supabase (leases) or review what’s on file under Tenants. The ⋮ menu shows the Active lease block when the room is occupied and a lease row exists.

Mark vacant or maintenance

  • Occupied card: primary button Mark vacant, or ⋮ → Set status → Vacant / Maintenance.
  • Maintenance card: Mark vacant or use the menu for any status.

Lease details & status (⋮ menu)

  1. Click ⋮ on a room card.
  2. Read Active lease (tenant, rent, advance/deposit, due day, next due, dates, notes) when applicable.
  3. Use Set status: Vacant / Occupied / Maintenance.

Flow 2 — Payments

  1. Open Payments.
  2. Review Total collected, This month, Pending, and the By billing month chart (paid vs pending by YYYY-MM).
  3. Add payment: room, amount (PHP), billing month, due date, optional paid date and notes → Save payment.
  4. Mark paid on a pending row sets status to paid and paid date to today.
  5. Data: Supabase payments (migration may seed pending rows for active leases for the current month).

Flow 3 — Tenant history

  1. Open Tenants.
  2. Scan Active vs ended leases, rent, due day, start/end, notes.
  3. Edits today: Supabase Table Editor (no in-app lease form yet).

Flow 4 — Utilities

  1. Open Utilities.
  2. Choose room, kind (electric / water), period, previous & current readings, optional notes → Save reading.
  3. Stored in utility_readings.

Status colors (dashboard cards)

Status Meaning
Vacant Available — green accent.
Occupied Rented — red accent.
Maintenance Not bookable — amber accent.

Prototype vs app (prototype-single.html)

The standalone prototype includes extras (invoice line items, guest inquiries, richer settlement math). Those are not in the Next.js app. Keep the prototype file as a visual archive if you want to rebuild features later.

Tech notes

  • RLS: Operational tables use authenticated policies (see supabase/migrations).
  • Room updates revalidate dashboard and related paths; payments actions revalidate the payments page.
